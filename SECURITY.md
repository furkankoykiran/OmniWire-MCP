# Security Policy

## Supported Versions

Currently, the latest version of OmniWire-MCP is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of OmniWire-MCP seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1.  **Do not open a GitHub issue.** Security issues should be reported privately to ensure that we have time to fix the issue before it's publicly disclosed.
2.  Send an email to [INSERT EMAIL ADDRESS] with the subject line "SECURITY VULNERABILITY".
3.  Provide a detailed description of the vulnerability, including steps to reproduce it if possible.
4.  Specify the impact of the vulnerability and any potential workarounds.

We will acknowledge receipt of your report within 48 hours and provide a timeline for addressing the issue. We appreciate your cooperation in helping us keep OmniWire-MCP secure.

## Process

After a vulnerability is reported, we will:

1.  Investigate and confirm the vulnerability.
2.  Develop a fix or mitigation.
3.  Release a new version with the fix.
4.  Publicly disclose the vulnerability and the fix in our release notes.

Thank you for helping us keep the community safe!
